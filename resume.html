  1. 主要负责物联网PaaS平台、技术中台、开放平台、视频中台和边缘服务的架构设计和实现落地的工作，实现应用支撑、设备管理、连接管理，数据管理，协议适配和对外开放的能力，满足各业务系统的对设备接入、控制和数据获取的需求；

  4. 实现统一API、适配多协议、多种视频流格式的物联网视频中台，支持GB28181（SIP协议）、Onvif、RTSP、海康、大华等标准或私有协议，熟悉ZLMediaKit 与 SRS 流媒体服务和 JavaCV与FFmpeg 流媒体处理框架，能够基于开源WVP-PRO进行扩展或者二次开发；
  5. 选用Axon Framework 实现物联网消息驱动架构，解决了消息分发、事件订阅、命令处理和数据查询响应的高并发性能、代码可扩展性、聚合对象管理、事件存储溯源、架构复杂度等问题；

  8. 使用JFR、JMC、MAT等应用监控、内存分析等方法手段进行应用问题诊断修复、代码优化和性能调优，解决应用的稳定性（如OOM）和性能（如死锁）等问题，促进性能、稳定性、代码质量的提升。

物联网 智能家居平台的架构设计与业务实现 分布式
  1. 基于NIO网络框架Netty开发支持集群与单机模式部署的TCP Broker，灵活扩展私有协议编解码器和数据帧等处理器解决消息粘包和编解码等问题，由TCP Broker注册中心负责集群管理和连接分配，解决设备就近连接、集群间负载均衡、集群探活与管理等问题；
  2. 基于 Spring Security + OAuth2 + JWT 实现智能家居平台对外开放的授权与验证，以及实现规范的、标准的、安全的对接第三方平台，使智能家居设备可以跨平台互联互通，相互控制，提升用户体验；
  3. 使用Nacos作为服务管理和配置管理中心，实现服务弹性伸缩、手动上下线管理、服务配置管理与配置动态刷新，解决微服务架构下服务注册发现、服务管理、配置管理分发等问题；
  4. 选用RabbitMQ作为消息队列中间件，基于Topic和Header等规则实现消息路由与分发，实现集群间通信和异步事件广播，解决智能家居设备事件消息生产、路由、消费、峰谷、丢失等问题；
  5. 选用开源规则引擎Easy-Rules，基于MVEL表达式实现智能家居事件处理、场景自动化、告警匹配、设备联动等需求，且轻量、可扩展、性能高、复杂度低、稳定可靠；
  6. 选用时序数据库InfluxDB，实现IOT大量实时消息的高性能的、多写少读的、无结构化的、时序性的消息持久化，解决消息记录的时序性和持久性问题，满足时序消息索引、高效查询、权限管理、时效管理、数据采样、压缩存储等需求，且HTTP接入简单无依赖；
  7. 熟悉对接百度DuerOS、天猫AliGenie、小米IOT、若琪Rokid、出门问问Mobvoi、猎豹Orion 、京东JDAlpha、科大讯飞AIUI、思必驰DUI 、Google Actions、Amazon Alexa等国内外AI开放平台、智能音箱平台和智能家居平台；
  8. 了解公有云Serverless服务，如AWS Lambda 请求流转发云函数和智能家居技能，实现智能家居平台对接亚马逊云平台的需求，使得Alexa可以控制智能家居设备，提升用户体验。
